# 4Sum

## Intuition

We need to find all unique quadruplets such that:

```text
nums[i] + nums[j] + nums[a] + nums[b] = target
```

Brute force would check every possible quadruplet:

```text
O(n⁴)
```

which is too slow.

👉 Sort the array and use the Two Pointer technique.

This is essentially an extension of the 3Sum problem.

---

## Key Insight

After sorting:

- Fix the first number (`i`)
- Fix the second number (`j`)
- Use two pointers (`a`, `b`) to find the remaining two numbers

This reduces complexity from:

```text
O(n⁴)
```

to:

```text
O(n³)
```

---

## Approach

### 1. Sort the Array

```cpp
sort(nums.begin(), nums.end());
```

Sorting allows us to use the two-pointer technique.

---

### 2. Fix First Element

Use:

```cpp
nums[i]
```

Skip duplicates:

```cpp
if (i > 0 && nums[i] == nums[i - 1])
    continue;
```

---

### 3. Fix Second Element

Use:

```cpp
nums[j]
```

Skip duplicates:

```cpp
if (j > i + 1 && nums[j] == nums[j - 1])
    continue;
```

---

### 4. Use Two Pointers

Initialize:

```cpp
a = j + 1;
b = n - 1;
```

Calculate:

```cpp
sum = nums[i] + nums[j] + nums[a] + nums[b];
```

---

### 5. Move Pointers

If:

```text
sum < target
```

Move:

```cpp
a++;
```

If:

```text
sum > target
```

Move:

```cpp
b--;
```

If:

```text
sum == target
```

Store quadruplet and move both pointers.

---

### 6. Skip Duplicates

After finding a valid quadruplet:

```cpp
while (a < b && nums[a] == nums[a - 1])
    a++;

while (a < b && nums[b] == nums[b + 1])
    b--;
```

This prevents duplicate answers.

---

## Complexity

- **Time:** `O(n³)`
- **Space:** `O(1)` (excluding answer array)

---

## Code

```cpp
vector<vector<int>> fourSum(vector<int>& nums, int target) {

    int n = nums.size();

    vector<vector<int>> ans;

    sort(nums.begin(), nums.end());

    for (int i = 0; i < n; i++) {

        if (i > 0 && nums[i] == nums[i - 1])
            continue;

        for (int j = i + 1; j < n; j++) {


            int a = j + 1;
            int b = n - 1;

            while (a < b) {

                long long sum =
                    (long long)nums[i] +
                    (long long)nums[j] +
                    (long long)nums[a] +
                    (long long)nums[b];

                if (sum < target) {
                    a++;
                }
                else if (sum > target) {
                    b--;
                }
                else {

                    ans.push_back(
                        {nums[i], nums[j], nums[a], nums[b]}
                    );

                    a++;
                    b--;

                    while (a < b &&
                           nums[a] == nums[a - 1]) {
                        a++;
                    }

                    while (a < b &&
                           nums[b] == nums[b + 1]) {
                        b--;
                    }
                }
            }
        }
    }

    return ans;
}
```

---

## Why Use long long?

For inputs like:

```cpp
1000000000
1000000000
1000000000
1000000000
```

Adding four integers can overflow `int`.

Use:

```cpp
long long sum
```

to avoid overflow.

## Example

Input:

```text
nums = [1,0,-1,0,-2,2]
target = 0
```

Sorted:

```text
[-2,-1,0,0,1,2]
```

Answer:

```text
[-2,-1,1,2]
[-2,0,0,2]
[-1,0,0,1]
```

## Pattern

```text
Sorting + Two Pointers

4Sum = 3Sum + One Extra Loop
```

---

## Common Mistakes

- Using brute force `O(n⁴)` ❌
- Forgetting to sort ❌
- Not skipping duplicates for `i` ❌
- Not skipping duplicates for `j` ❌
- Not skipping duplicates for pointers ❌
- Using `int` and causing overflow ❌

---

## Memory Trick

```text
Fix i
Fix j

Use Two Pointers

Too Small → Left++
Too Large → Right--
Equal → Store Answer
```

---

## Relation to 3Sum

```text
2Sum → One Loop + Two Pointers

3Sum → Two Loops + Two Pointers

4Sum → Three Loops (i,j) + Two Pointers
```
